FP-005 — OpenCode Telemetry Ingestion

Task

Implement authoritative OpenCode telemetry ingestion for ForgePilot packet execution records.

Goal

Replace null execution telemetry fields with telemetry gathered from OpenCode session artifacts, while preserving ForgePilot’s trust, authority, and persistence standards.

FP-005 answers one implementation question:

How does ForgePilot ingest authoritative model execution telemetry without allowing executors to self-report cost or token usage?

This packet implements telemetry ingestion only.

It does not add model routing, autonomous execution, broker behavior, scoring, aggregation, dashboards, reporting, or routing decisions.

Governing Principles

This packet is constrained by:

* P01 — ForgePilot records observations, not narratives.
* P02 — Trust cannot be retroactively created.
* P03 — ForgePilot does not optimize for favorable outcomes.
* P04 — Only admitted evidence may influence observatory outputs.
* P05 — Do not build infrastructure for evidence that does not yet exist.
* P06 — Classification follows observation.

Prior Standards

Implementation must follow:

* docs/metrics-trust-and-validation.md
* docs/telemetry-authority-and-field-ownership.md
* docs/persistence-standards.md
* docs/model-evaluation-harness.md
* packets/FP-004.md

FP-005 must not redefine those standards.

Authority Rule

Executors must not author their own cost or token telemetry.

OpenCode session data is the authoritative source for:

* OpenCode session ID
* provider
* model
* model variant, if present
* input tokens
* output tokens
* reasoning tokens
* cache read tokens
* cache write tokens
* cost
* session created timestamp
* session updated timestamp
* duration, if derivable

Model-written files may reference telemetry only as untrusted notes unless the value is backed by OpenCode telemetry.

Required Input Model

Implement ingestion from an OpenCode session export artifact.

The ingestion input must be a local JSON file produced by OpenCode export or equivalent saved OpenCode session data.

FP-005 must not require live network access.

FP-005 must not require calling a provider API.

FP-005 must not require invoking OpenCode itself.

Required Telemetry Fields

The ingestion logic must extract, when present:

* session_id
* provider
* model
* model_variant
* input_tokens
* output_tokens
* reasoning_tokens
* cache_read_tokens
* cache_write_tokens
* cost
* created_at
* updated_at
* duration_ms

If a field is unavailable, it must remain null.

Unavailable fields must not be guessed.

Required Mapping Fields

Telemetry ingestion must require explicit mapping to a ForgePilot execution attempt.

Required mapping inputs:

* packet_id
* execution_id
* telemetry_artifact_path

Optional mapping inputs:

* run_branch
* executor_model
* notes

The ingestion code must not infer execution mapping from session title alone.

Session title may be recorded as supporting metadata, but must not be the sole mapping key.

Required Persistence Behavior

Update the FP-004 SQLite persistence model to store OpenCode telemetry for execution attempts.

The implementation may either:

1. Add telemetry columns to packet_executions, or
2. Add a separate execution telemetry table linked by execution_id.

Preferred implementation:

packet_execution_telemetry
  one row per execution_id per telemetry source artifact

Required fields if using a separate table:

* telemetry_id
* execution_id
* packet_id
* source
* telemetry_artifact_path
* opencode_session_id
* provider
* model
* model_variant
* input_tokens
* output_tokens
* reasoning_tokens
* cache_read_tokens
* cache_write_tokens
* cost
* session_created_at
* session_updated_at
* duration_ms
* ingestion_mode
* mapping_confidence
* recorded_at

Required values:

source = OPENCODE_TELEMETRY
ingestion_mode = DIRECT_ARTIFACT | RETROACTIVE_ARTIFACT
mapping_confidence = EXPLICIT

Retroactive Ingestion

FP-005 may support retroactive telemetry ingestion from previously exported OpenCode sessions.

Retroactive ingestion must be marked explicitly:

ingestion_mode = RETROACTIVE_ARTIFACT

Retroactive ingestion must not rewrite historical executor artifacts.

Retroactive ingestion must not pretend the telemetry was captured during the original run.

Trust and Admission Behavior

Telemetry ingested from OpenCode artifacts must be classified as:

source = OPENCODE_TELEMETRY
trust_tier = TIER_2_VERIFIED_ARTIFACT
validation_state = VALID if required mapping is explicit and artifact parses successfully
validation_state = INCOMPLETE if telemetry parses but required fields are missing
admission_state = PENDING by default

FP-005 must not automatically admit telemetry into observatory outputs.

Admission remains a separate validation/audit concern.

Code Requirements

Add or update code that supports:

1. Parsing a local OpenCode telemetry JSON artifact.
2. Extracting required telemetry fields.
3. Handling missing telemetry fields as null.
4. Requiring explicit packet_id and execution_id mapping.
5. Persisting telemetry to SQLite.
6. Supporting direct artifact ingestion.
7. Supporting retroactive artifact ingestion.
8. Preventing executor-authored metrics from being treated as authoritative telemetry.
9. Testing successful telemetry ingestion.
10. Testing missing-field behavior.
11. Testing explicit mapping requirement.
12. Testing retroactive ingestion labeling.
13. Testing idempotent migration/init behavior.

CLI Requirements

Add a small local CLI command for ingestion.

Suggested command:

pnpm fp -- ingest-opencode-telemetry \
  --packet-id FP-004 \
  --execution-id <execution_id> \
  --artifact-path <path-to-opencode-export.json> \
  --mode retroactive

The exact command name may differ if it fits existing CLI style better, but it must remain local and non-networked.

The command must fail clearly if:

* artifact path is missing
* artifact JSON cannot be parsed
* packet_id is missing
* execution_id is missing
* execution_id does not exist in SQLite
* mode is not direct or retroactive

Acceptance Criteria

The implementation is accepted only if:

* OpenCode telemetry artifact parsing is implemented.
* Telemetry comes from a local artifact, not executor self-reporting.
* No live provider or OpenCode API calls are required.
* Explicit packet_id and execution_id mapping is required.
* Telemetry is persisted in SQLite.
* Missing telemetry fields remain null.
* Cost and token values are not guessed.
* Retroactive ingestion is labeled distinctly.
* Telemetry source is recorded as OPENCODE_TELEMETRY.
* Migration/init behavior remains idempotent.
* Existing FP-004 persistence behavior remains working.
* Existing tests pass.
* New tests cover telemetry ingestion behavior.
* No routing, broker behavior, autonomous execution, aggregation, reporting, dashboards, scoring, or automatic admission is added.

Rejection Criteria

Reject the implementation if it:

* Allows executors to self-report authoritative token or cost telemetry.
* Infers execution mapping from session title alone.
* Requires network access.
* Calls provider APIs.
* Calls live OpenCode APIs.
* Estimates missing token or cost values.
* Rewrites old run artifacts as source of truth.
* Automatically admits telemetry.
* Breaks FP-004 persistence behavior.
* Adds routing, broker behavior, autonomous execution, aggregation, reporting, or dashboards.
